> 版本: v1.4 | 日期: 2026-06-08 | 状态: v1.4 已交付

## 1. 项目背景

团队约 20 人，日常依赖 Wiki 和 Markdown 文档进行工作。本地文档和外部网页分散在不同系统，缺乏统一检索入口。现有搜索系统需要更精准的关键词匹配和排序能力。

### 3.2 F2 混合搜索详细需求

**排序算法:**
- TF-IDF char_wb 余弦相似度 → 排名
- BM25 关键词匹配（词频饱和 + 长度归一化 + IDF）→ 排名
- RRF（倒数排名融合）合成最终排序
- 相关度归一化到 0~1 区间

**分词策略:**
- 中文：单字切分（"开发环境" → "开","发","环","境"）
- 英文：整词保留（"VS Code" → "vs","code"）

**多层过滤:**
1. TF-IDF 绝对阈值：max raw TF-IDF < 0.05 → 无结果
2. BM25 零检查：max BM25 == 0 → 无结果（词汇不存在）
3. Dense ranking：同分同名次，零分块强制末位
4. 相关度阈值：RRF < 0.80 → 排除

## 7. 约束与假设

| 约束 | 说明 |
|------|------|
| 文档格式 | 本地 .md；外部网页通过 HTML→MD 转换 |

## 8. 术语表

| 术语 | 说明 |
|------|------|
| BM25 | Best Matching 25，概率检索模型，对词频饱和和文档长度归一化 |
| RRF | Reciprocal Rank Fusion，倒数排名融合，多排序信号的无权重融合方法 |
| Dense Ranking | 稠密排名：同分同名次（1,2,2,3...） |
| TF-IDF | 词频-逆文档频率 |
| RRF 归一化 | RRF 原始分 / 理论最大值 → 0~1 |
