# 知识库搜索系统 — 需求文档

> 版本: v1.2 | 日期: 2026-06-07 | 状态: MVP 增强版已交付

---

## 1. 项目背景

团队约 20 人，日常依赖 Wiki 和 Markdown 文档进行工作。当前查找信息的方式是手动翻阅文档目录或 Wiki 页面，效率低下，尤其对新人而言。

**核心痛点:**

- 新人入职后不知道该查哪个文档、在哪里查
- 每个成员每天查文档 3-4 次，全团队每天约 60-80 次查询
- 每次手动翻找需耗时 2-5 分钟
- 文档分散在不同目录，缺乏统一检索入口

## 2. 项目目标

构建一个内部知识库搜索系统，让团队成员通过关键词快速找到对应的文档内容。分阶段交付：

| 阶段 | 目标 | 状态 |
|------|------|------|
| MVP | CLI + Web API 搜索本地 Markdown 文档 | ✅ 已完成 |
| MVP 增强 | Web 搜索页面（浏览器输入框搜索） | ✅ 已完成 |
| Phase 2 | 接入企业微信，在企业微信内直接搜索 | 🔲 待实施 |
| Phase 3 | 混合搜索优化（向量 + 关键词）、新人 FAQ 专区 | 🔲 规划中 |

**成功指标:**

| 指标 | 目标值 | 当前值 |
|------|--------|--------|
| 搜索响应时间 | < 1 秒 | < 100ms (CLI), < 200ms (Web) |
| 搜索结果 Top-3 命中率 | > 80% | 待验证 |
| 新人上手时间 | < 5 分钟 | ✅ 开箱即用 |
| 系统可用性 | 99% (工作时间) | ✅ 单机无外部依赖 |

### 4.2 F6 Web 搜索页面详细需求

**页面布局:**
- 顶部蓝色 Header：系统标题"📚 知识库搜索" + 副标题
- 搜索栏：输入框 + "搜索"按钮，输入框自动聚焦
- 结果区：卡片式展示，每条结果包含标题、文件路径、内容摘要、相关度分数

**交互行为:**
- 输入关键词后点击搜索或按 Enter 发起搜索
- 搜索中显示"搜索中..."加载状态
- 无结果时显示空状态提示"😕 未找到与「xxx」相关的文档"
- 出错时显示错误信息

**视觉设计:**
- 简洁现代风格，蓝色主题 (#1890ff)
- 搜索结果卡片悬停时阴影加深
- 响应式布局，适配移动端

## 7. 约束与假设

| 约束 | 说明 |
|------|------|
| 文档格式 | 仅支持 Markdown (.md) 文件 |
| 文档位置 | 必须在 `docs/` 目录下，支持子目录递归 |
| 索引更新 | 文档变更后需手动运行 `index_docs.py` 重建索引 |
| 部署环境 | 单机部署，不支持分布式 |
| 字符编码 | 文档需 UTF-8 编码 |
| 用户规模 | 20 人团队，并发 < 5 请求/秒 |

## 8. 术语表

| 术语 | 说明 |
|------|------|
| 文本块 (Chunk) | 从文档中切分的检索单元，最大 500 字符 |
| TF-IDF | Term Frequency-Inverse Document Frequency，词频-逆文档频率，衡量词语在文档中的重要程度 |
| char_wb | Character-level Word Boundary 分析器，在词边界内做字符 n-gram |
| 余弦相似度 | 衡量两个向量方向相似程度的指标，用于计算查询与文档的相关性 |
| 混合排序 | TF-IDF 相似度 + 关键词命中加权的综合排序算法 |
