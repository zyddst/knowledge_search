## 2. 技术选型

| 组件 | 方案 | 选型原因 |
|------|------|----------|
| 编程语言 | Python 3.11 | 团队熟悉，生态丰富 |
| 检索引擎 | scikit-learn TfidfVectorizer | 纯 Python，零原生依赖，Windows 无痛部署 |
| 分块策略 | 段落分块 + char_wb 分词 | 中文友好，n-gram 兼顾精确关键词和模糊匹配 |
| 排序算法 | TF-IDF 余弦相似度 + 关键词命中加权 | 简单有效，20 人规模足够 |
| 序列化 | pickle | Python 原生，读写速度快 |
| Web 框架 | http.server (标准库) | 零额外依赖，MVP 场景够用 |
| 前端 | 内联 HTML + 原生 JS | 单文件分发，零构建步骤 |
| 网页抓取 | requests + BeautifulSoup4 | Python 生态标准，HTML 解析和 CSS 选择器支持 |
| 企微加解密 | pycryptodome | AES-256-CBC 加解密，企业微信加密模式必须 |

## 3. 模块设计

### 3.1 项目结构

```
project/
├── docs/                    # 本地 Markdown 文档（用户手动维护）
│   ├── 新人指南.md
│   └── API接口规范.md
├── fetched_docs/            # 外部文档缓存（fetcher.py 自动生成）
│   └── <source_name>/
│       └── *.md
├── specs/                   # 项目规划文档（不被索引搜索）
│   ├── 需求文档.md
│   └── 系统设计文档.md
├── vector_db/               # 索引文件（自动生成）
│   ├── chunks.pkl           # 文本块列表 [{id, content, source, title}, ...]
│   ├── vectorizer.pkl       # TfidfVectorizer 实例
│   ├── tfidf_matrix.pkl     # TF-IDF 稀疏矩阵 (n_chunks × n_features)
│   └── meta.txt             # 元信息（索引源、文件数、块数）
├── config.py                # 全局配置（含外部文档源）
├── fetcher.py               # 网页抓取 + HTML → Markdown 转换
├── index_docs.py            # 索引构建（多源：docs/ + fetched_docs/）
├── search.py                # CLI 搜索模块
├── server.py                # Web 服务模块（API + 搜索页面）
├── wechat.py                # 企业微信加密消息处理
├── requirements.txt         # 依赖声明
└── README.md                # 项目说明
```

### 3.2 config.py — 配置模块

```python
# 可调参数
DOCS_DIR = "./docs"                # 本地文档目录
FETCHED_DOCS_DIR = "./fetched_docs" # 外部文档缓存目录
VECTOR_DB_DIR = "./vector_db"       # 索引存储路径
CHUNK_SIZE = 500                    # 文本块大小（字符数）
CHUNK_OVERLAP = 50                  # 滑动窗口重叠量
TOP_K = 5                           # 返回结果数

# 外部文档源
EXTERNAL_SOURCES = [
    {
        "name": "Python 教程",
        "urls": ["https://docs.python.org/zh-cn/3/tutorial/introduction.html"],
        "content_selector": "article",
        "exclude_selectors": [".sidebar", ".headerlink"],
    },
]
```

### 3.3 fetcher.py — 网页抓取模块

**职责:** 根据 `EXTERNAL_SOURCES` 配置抓取外部网页，提取正文并转换为 Markdown，缓存到 `fetched_docs/`。

**核心函数:**

| 函数 | 输入 | 输出 | 说明 |
|------|------|------|------|
| `fetch_page(url, selector, exclude)` | URL + CSS 选择器 | `str` | 抓取单页面，提取正文，返回 Markdown |
| `fetch_source(source, dir)` | 源配置 + 输出目录 | `int` | 抓取一个源的所有 URL，保存为 .md |
| `fetch_all(no_fetch)` | 是否跳过抓取 | `int` | 遍历所有外部源，缓存到本地 |

**HTML → Markdown 转换策略:**

```
网页 URL
  │
  ├── requests.get() → HTML
  ├── BeautifulSoup 解析
  │     ├── 移除 script / style / nav / footer / header
  │     ├── 移除 exclude_selectors 指定的元素
  │     └── 提取 content_selector 指定的正文区域
  │           或 auto: article > main > body
  │
  ├── 递归提取 (_extract_text):
  │     ├── h1-h6 → # 标题
  │     ├── p     → 段落
  │     ├── a     → [text](href)
  │     ├── li    → - 列表
  │     ├── pre/code → ``` 代码块
  │     ├── img   → ![alt](src)
  │     └── table → Markdown 表格
  │
  └── 保存为 .md 文件（UTF-8）
      ├── 插入标题行: # 页面 title
      ├── 插入元信息: > 来源: URL \n > 抓取时间: ...
      └── 插入正文内容
```

### 3.4 index_docs.py — 索引构建模块

**职责:** 扫描多个文档源 → 分块 → TF-IDF 向量化 → 持久化

支持三种模式：
- `python index_docs.py` → 扫描 docs/ + fetched_docs/
- `python index_docs.py --fetch` → 先抓取外部文档，再索引
- `python index_docs.py /path/to/custom` → 索引指定目录（兼容旧用法）

**核心函数:**

| 函数 | 说明 |
|------|------|
| `collect_markdown_files(dirs)` | 从多个目录收集 .md 文件，跨目录去重 |
| `chunk_markdown(filepath)` | 按段落分块，滑动窗口切分超长段落 |
| `build_index(dirs, do_fetch)` | 主流程：可选抓取 → 收集 → 分块 → TF-IDF → 持久化 |

**分块策略：**

```
Markdown 文件
  │
  ├── 解析标题（第一个 # 作为文档标题）
  ├── 按空行分割为段落
  ├── 合并短段落直到接近 CHUNK_SIZE (500 chars)
  ├── 超长段落用滑动窗口切分（overlap=50 chars）
  │
  └── 每个块生成唯一 ID: md5(filepath:chunk_index)[:12]
```

**TF-IDF 参数:**

```python
TfidfVectorizer(
    analyzer="char_wb",     # 字符级 n-gram，词边界内
    ngram_range=(2, 4),     # 2-4 gram，兼顾短词和长词
    max_features=10000,     # 词汇表上限
)
```

### 3.5 search.py — CLI 搜索模块

**混合排序算法:**

```
score(chunk, query) = TF-IDF_cosine(query, chunk)
                    + keyword_hits * 0.05    # 关键词在正文命中数
                    + title_hits * 0.2       # 关键词在标题命中数
```

**去重策略:** 同一源文件只保留得分最高的一个块。

## 4. 数据流

### 4.1 索引构建流程（含外部文档抓取）

```
用户运行: python index_docs.py --fetch
          │
          ├── [0/4] 抓取外部文档 (fetcher.fetch_all)
          │     ├── 遍历 EXTERNAL_SOURCES 配置
          │     ├── 每个 URL → requests.get() → HTML
          │     ├── BeautifulSoup 提取正文 → Markdown
          │     └── 保存到 fetched_docs/<source_name>/*.md
          │
          ├── [1/4] 收集所有 .md 文件 (collect_markdown_files)
          │     ├── docs/ 下的本地文档
          │     └── fetched_docs/ 下的外部文档（如果存在）
          │
          ├── [2/4] chunk_markdown() 对每个文件
          │     ├── 解析标题 → 段落分割 → 滑动窗口
          │     └── 每个块附 {id, content, source, title}
          │
          ├── [3/4] TfidfVectorizer.fit_transform(all_contents)
          │     ├── 学习 n-gram 词汇表
          │     └── 生成 TF-IDF 稀疏矩阵
          │
          └── [4/4] pickle.dump → vector_db/
                ├── chunks.pkl, vectorizer.pkl, tfidf_matrix.pkl
                └── meta.txt
```

### 4.2 搜索流程

```
用户入口:
  ├── 浏览器: http://localhost:8080 → 输入框 → JS fetch /search?q=xxx
  ├── CLI: python search.py "关键词"
  └── API: curl localhost:8080/search?q=xxx
          │
          ▼
    search.py / server.py → search(query)
          │
          ├── 加载 vector_db/ 中的索引（首次加载后常驻内存）
          │
          ▼
    vectorizer.transform(query) → TF-IDF 向量
          │
          ▼
    cosine_similarity(query_vec, tfidf_matrix) → 相似度 [N]
          │
          ▼
    关键词提取 + 加权 → 综合排序 → 去重 → Top-K
          │
          ▼
    输出: [{title, source, snippet, score}, ...]
    来源可以是 docs/ 或 fetched_docs/ 下的任何文件
```

## 5. 部署说明

### 环境要求

- Python 3.8+
- scikit-learn >= 0.24, requests, beautifulsoup4, pycryptodome

### 启动步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置外部文档源（可选）
# 编辑 config.py 中的 EXTERNAL_SOURCES

# 3. 构建索引（含抓取外部文档）
python index_docs.py --fetch

# 4. 搜索
python search.py "关键词"

# 5. 启动 Web 服务
python server.py
```

### 文档更新

```bash
# 本地文档变更
python index_docs.py

# 外部文档重新抓取 + 索引
python index_docs.py --fetch
```

## 7. 升级路线

### 当前: TF-IDF + 多源索引 (v1.3)

- 适用: < 100 篇文档，含外部网页
- 优点: 本地+外部文档统一搜索
- 局限: 仅关键词匹配，无语义理解

### 下一步: 向量嵌入 (v2.0)

- 模型: bge-large-zh 或 text2vec-base-chinese
- 向量库: ChromaDB 或 Milvus Lite
- 检索: 混合搜索（向量相似度 + BM25 关键词）

### 远期: RAG + 答案生成 (v3.0)

- 搜索结果送给 LLM 生成自然语言答案

## 8. 已知限制

| 限制 | 影响 | 缓解方案 |
|------|------|----------|
| 仅支持 .md 文件 | Word/PDF 文档需先转换 | 后续加 `python-docx` / `pypdf` 支持 |
| 全量索引重建 | 文档多时耗时长 | 后续加增量索引 |
| 无语义理解 | "报销"搜不到"费用申请" | v2.0 向量嵌入方案 |
| 网页抓取需联网 | 离线时 fetch 失败 | `--no-fetch` 跳过抓取，使用缓存 |
| 单机部署 | 无法多人在不同机器访问 | server.py 可在局域网访问 |
| 无认证 | API 无鉴权 | 内网场景可接受 |

## 9. 企业微信接入设计（开发中）

### 架构

```
企业微信服务器
  │
  ├── GET  /wechat?echostr=...  → URL 验证（签名校验 + AES 解密）
  └── POST /wechat  <xml>       → 消息接收（解密 → 搜索 → 加密 → 回复）
```

### 消息处理模块 (wechat.py)

| 函数 | 功能 |
|------|------|
| `calc_signature()` | SHA1 签名计算 |
| `decrypt(enc, key)` | AES-256-CBC 解密 |
| `encrypt(msg, key, corp_id)` | AES-256-CBC 加密 |
| `parse_encrypted_message(xml, key)` | 解析加密 XML → 明文消息 dict |
| `build_encrypted_reply(...)` | 构建加密回复 XML |
| `format_search_reply(query, results)` | 搜索结果 → 企微聊天文本 |

### 配置项

```python
WECHAT_CORP_ID = "ww..."           # 企业 ID
WECHAT_TOKEN = "your_token"        # 回调 Token
WECHAT_ENCODING_AES_KEY = "..."    # 43 字符 Base64 密钥
```
