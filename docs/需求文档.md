# 知识库搜索系统 — 需求文档

> 版本: v1.2 | 日期: 2026-06-07 | 状态: MVP 增强版已交付

## 1. 项目背景

团队约 20 人，日常依赖 Wiki 和 Markdown 文档进行工作。当前查找信息的方式是手动翻阅文档目录或 Wiki 页面，效率低下，尤其对新人而言。

**核心痛点:** 新人入职后不知道该查哪个文档、在哪里查。每个成员每天查文档 3-4 次，全团队每天约 60-80 次查询，每次手动翻找耗时数分钟。

## 2. 项目目标

构建一个内部知识库搜索系统，让团队成员通过关键词快速找到对应的文档或 wiki 内容。分阶段交付：

| 阶段 | 目标 | 状态 |
|------|------|------|
| MVP | CLI + Web API 搜索本地 Markdown 文档 | ✅ 已完成 |
| MVP 增强 | Web 搜索页面（浏览器输入框搜索） | ✅ 已完成 |
| Phase 2 | 接入企业微信，在企业微信内直接搜索 | 🔲 待实施 |
| Phase 3 | 混合搜索优化（向量 + 关键词）、新人 FAQ 专区 | 🔲 规划中 |

## 3. 用户角色

| 角色 | 描述 | 典型场景 |
|------|------|----------|
| 普通成员 | 团队日常使用者 | "报销流程怎么走？""代码规范是什么？" |
| 新人 | 入职不足 1 个月 | "开发环境怎么搭？""账号怎么开通？" |
| 管理员 | 负责维护文档和索引 | 文档变更后重建索引 |

## 4. 功能需求

### 4.1 已完成功能

| 编号 | 功能 | 描述 | 版本 |
|------|------|------|------|
| F1 | 文档索引 | 扫描指定目录下所有 `.md` 文件，分块并建立 TF-IDF 索引 | v1.0 |
| F2 | 关键词搜索 | 输入自然语言关键词，返回相关文档片段，按相关度排序 | v1.0 |
| F3 | CLI 搜索 | 命令行工具：`python search.py "关键词"` | v1.0 |
| F4 | Web API | HTTP 服务，支持 GET/POST 搜索，返回 JSON | v1.0 |
| F5 | 企业微信回调 | `/wechat` 端点，接收 keyword 返回 reply | v1.0 |
| F6 | Web 搜索页面 | 浏览器访问 `http://localhost:8080`，提供搜索输入框和结果展示页面 | v1.1 |

### 4.2 后续功能（规划中）

| 编号 | 功能 | 描述 |
|------|------|------|
| F7 | 企业微信集成 | 在企业微信内 @机器人 搜索，直接返回结果 |
| F8 | 新人 FAQ 专区 | 高权重预设关键词（报销、入职、环境搭建），新人优先命中 |
| F9 | 增量索引 | 文档变更时自动重建对应部分索引 |
| F10 | 搜索结果溯源 | 返回结果带原文链接（文件路径 + 行号） |

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 性能 | 20 人、~100 篇文档场景下，搜索响应时间 < 1 秒 |
| 可用性 | 零外部依赖（不联网、不调 API），纯本地运行 |
| 兼容性 | Windows 终端 GBK 编码下稳定运行，中文输入/输出不乱码、不崩溃；启动时正常显示横幅信息 |
| 可维护性 | 新增文档只需放入 `docs/` 目录，运行 `index_docs.py` 即可 |
| 部署 | Windows 下开箱即用，仅需 Python 3.8+ 和 scikit-learn |
